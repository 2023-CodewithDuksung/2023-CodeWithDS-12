### 📚 커밋 컨밴션

| 커밋         | 역할                                                  |
| ------------ | ----------------------------------------------------- |
| [ init ]     | 작업 세팅 커밋 (패키지 설치 등)                       |
| [ feat ]     | 새로운 기능이 추가되었을 때                           |
| [ fix ]      | 버그를 고친 경우                                      |
| [ design ]   | css 혹은 UI를 생성 & 변경한 경우                      |
| [ style ]    | 코드 포맷 변경 (기능에 변화가 없는 경우)               |
| [ refactor ] | 더 좋은 코드로 개선한 경우 ( 기능에 변화가 없는 경우) |
| [ docs ]     | README 등 문서를 추가한 경우                          |
| [ test ]     | 테스트코드 작성                                       |
| [ etc ]      | 이 외 수정                                        |
